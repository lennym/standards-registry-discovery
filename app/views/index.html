Prototypes for joining up IT systems in health and social care

This is a collection of prototypes for the interoperability standards catalogue Alpha.


Round 5 - User research 6th - 10th December 2021 Focus: maintenance and contribution (second alpha)
Version notes

Changes include:

Homepage
  • Renamed headings to 'Standards directory and roadmap', 'Guidance on using standards' and 'Community and contributions'
  • Reduced the number of cards under each heading to two
  • Removed 'Browse by topic', 'Common use cases', 'Case studies', 'Share standard or API' and 'Identify a gap'
  • Added two new cards on roadmap and contribution guidance
Browse all standards
  • Replaced 'Browse directory' page with 'Browse all standards', thereby removing categories in the former and relying on filters for refinement in the latter
  • Removed 'Content standards', 'Technical standards' and 'Services' pages
  • Updated filters to 'Business use', 'Care settings', 'Standard type' and 'Status'
  • Updated the status labels to include Deprecated as the fourth category
Standard listing page
  • Changed the caption from 'Content format' to 'Information standard listing'
  • Grouped metadata into sections: 'About the standard', 'Business and care setting usage', 'Relationships to other standards', 'Assurance and endorsements'
  • Moved link to full standard documentation and standard owner's contact details further up on the page into 'About this standard'
  • Removed 'Previous versions', use cases, case studies
  • Renamed 'Technical specifications' into 'Related standards'
Roadmap
  • Set up new roadmap page outlining standards that are commissioned, drafted, newly active, to be deprecated, deprecated and retired
Health and social care standards community platform
  • Set up a new community platform using GitHub, an open source collaboration tool
  • Created an intro page explaining how the platform works and main contribution areas of the platform
  • Created 4 templates to help raise issues around proposals for new standards, identifying gaps in existing standards, suggesting improvements and reporting problems with the directory
  • Customised labels to help with tagging and filtering contributions
  • Set up a discussions forum to help provide guidance and support
  • Set up a standards community development backlog to manage contributions to standards development

Round 2 - User research 22nd to 23rd April 2021
Version notes

Changes include:

Homepage
  • It’s now called ‘NHS Standards directory’ rather than ‘Open toolkit’
  • We’ve added some examples for the search box
  • If you add a search term and hit submit, you’ll go to a very basic page of search results with some filters
  • The ‘Tools’ section is now called ‘Directory’
  • We’ve swapped the position of the Guidance and Directory links, so Directory now comes first - we think the links to the tools will be by far the more used section for most users, so we want to avoid frustrating them by forcing them to scroll down every time. But having the Guidance on the second row hopefully gives it enough prominence that users looking for it will be able to see it.
  • Some of the labels of the different sections have been tweaked
  • The ‘Contribute’ section how has 4 sections, rather than 2
Browse directory
  • We’ve combined the ‘Information standards’ and ‘Terminology standards’ sections into a single group called ‘Content Standards’
  • We’ve added a section for ‘Security standards’ - but this is a placeholder and doesn’t link through to anything – it could be worth probing what users think might be in here
Content standards
  • We’ve added some new filters for
  • ‘Compliance’ – I.e. is the standard ‘required’ ‘recommended’ or ‘optional’
  • ’Status’ - I.e. what is the status of the standard
  • These are both highlighted in the list of standards
  • We will also add in the date the standard was last updated
Standard page – mental health inpatient discharge
  • We’ve rearranged the metadata a bit
  • The link to the standard is now nearer the top of the page under the description
  • We’ve metadata for ‘compliance’ along with an explanation of what it means e.g. ‘Recommended for use on relevant services’ – it would be good to probe what they think of the description, and what would they think it should say if it was a mandated standard
  • We’ve added related use cases and case studies near the bottom
  • Below the ‘Previous versions’,
  • We’ve added a link to ‘Get email updates for this standard’ where the user can provide an email address
  • We’ve added a page last update section
  • We’ve added a was this page useful? link to help gather feedback
Browse technical standards
  • We’ve added an “API status’ filter with more simplified options, rather than ‘Alpha beta etc’– it would be good to probe if this is more understandable or it needs more detail
  • We play back the API status on the list
  • We play back the date last updated on the list
Browse by use case
  • Name has been changed from ‘Browse by product type’
  • The list of care settings has been reduced a bit by making some of the options (Hospital, Social Care and Urgent and emergency care) conditionally revealing
  • The content on the results page has been rearranged a bit
Common use cases
  • No major changes, we’ve added ‘page last updated’ and ‘Was this page useful
Guidance section: What is interop and get started
  • Both have new guidance
  • Both have ‘page last updated’ and ‘Was this page useful
Browse case studies
  • We’ve added a few more examples, although they all link to the Bradford example
  • We’ve updated the case study names to be more descriptive
  • We’ve added filters for ‘Region’ and “Technology
Bradford case study
  • We’ve added a link to download the case study
  • We’ve added ‘page last updated’ and ‘Was this page useful
Contribute
  • We have 4 sections now, but none link through

Round 1 - User research 14th to 16th April 2021
Version notes

Initial prototype developed to explore:

  • High level guidance sections
  • Case studies content
  • Navigation based on browsing by tools (Standards, APIs and services)
  • An example standards page for an infomration standard (mental health inpatient dishcarge standard)
  • An example standards page for a terminology standard (SNOMED CT)
  • An example standards page for a technology standard (Transfer of Care mental health dishcarge - FHIR API )
  • Navigation based on product type and care setting
  • Navigation based on service use cases
  • An example use case (Discharging someone from mental health services to another care setting)
  • What a use case could look like and cross link to all related standards
  • Page and information formats for information and terminology standards
